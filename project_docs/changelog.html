Changelog

Track all changes, updates, and improvements to the Claude Code Baseline repository.

Version 2.0 - Major Update

January 15, 2025

Major release introducing modular coding standards, existing project integration, and enhanced automation capabilities.

New Features
  • Add-to-Existing-Project Script
    Safely integrate baseline into existing projects with automatic backups, conflict detection, and rollback capabilities.
  • Modular Coding Standards
    Split monolithic coding standards into 13 focused, navigable documents with central README hub.
  • HTML Documentation Site
    Professional Bootstrap-based documentation site with responsive design and shared navigation components.
  • Enhanced Template Variables
    Expanded variable system with {{CLAUDE_WIP_PATH}}, {{DATE}}, {{YEAR}} for better automation.
  • 8 AI Agent Templates
    Pre-configured agents for documentation, code review, testing, security, refactoring, API design, database optimization, and troubleshooting.
Changes
  • Folder Rename: coding-standards/
    Renamed from 03-coding-standards/ to coding-standards/ for cleaner project structure.
  • Claude WIP Convention
    Formalized claude_wip/ as standard working directory for all AI-assisted development across projects.
  • Updated Documentation
    Enhanced CLAUDE.md, README.md, and all baseline templates with latest best practices and clearer instructions.
Improvements
  • Better error handling in PowerShell scripts
  • Comprehensive logging and reporting
  • Dry-run mode for all scripts
  • Git integration detection
  • Automatic verification checks
Migration Guide: Existing projects can safely upgrade using add-baseline-to-existing-project.ps1 with the -DryRun flag to preview changes first.

Upgrade Instructions

Upgrading from v1.x to v2.0
Important: Always create a backup before upgrading. Version 2.0 includes structural changes.
For New Projects:
# Simply use the latest new-project script
.\new-project.ps1 -ProjectName "MyApp" -DestinationPath "E:\projects\myapp"
For Existing Projects:
# Step 1: Preview changes (dry-run)
.\add-baseline-to-existing-project.ps1 -ProjectPath "E:\myproject" -DryRun

# Step 2: Apply changes
.\add-baseline-to-existing-project.ps1 -ProjectPath "E:\myproject"

# Step 3: Manually rename coding standards folder if present
# From: docs\03-coding-standards\
# To:   docs\coding-standards\
Breaking Changes:
  • Folder renamed: 03-coding-standards/ ‚Üí coding-standards/
  • Coding standards split into 13 files (previously 1 monolithic file)
  • New required directory: claude_wip/ with subdirectories
Compatible Changes:
  • All baseline documentation templates remain compatible
  • Template variable system is backward compatible
  • PowerShell scripts have improved error handling but same interface
