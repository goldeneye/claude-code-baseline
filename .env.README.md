## Critical Reminders

### ‚ùå NEVER Do These:

1. **NEVER remove `.env` from `.gitignore`**
2. **NEVER run `git add .env` or `git add -f .env`**
3. **NEVER share `.env` file via:**
   - Email
   - Slack/Discord
   - Screen sharing (hide it!)
   - Copy/paste
   - USB drives without encryption

### ‚úÖ ALWAYS Do These:

1. **ALWAYS verify before commits:**
   ```bash
   git status
   # .env should NOT appear in the output
   ```

2. **ALWAYS check before pushing:**
   ```bash
   git diff --cached
   # .env should NOT appear in the diff
   ```

3. **ALWAYS use `.env.example` for team sharing:**
   - `.env.example` contains placeholders only
   - Safe to commit and share
   - Team members copy to create their own `.env`

## If .env is Accidentally Committed

**IMMEDIATE ACTIONS:**

1. **DO NOT just remove from Git** - it stays in history!

2. **Rotate ALL API keys immediately:**
   - Anthropic: https://console.anthropic.com/settings/keys
   - Any other services

3. **Remove from Git history:**
   ```bash
   # Use git-filter-repo or BFG Repo-Cleaner
   git filter-repo --invert-paths --path .env
   ```

4. **Force push to remote (if already pushed):**
   ```bash
   git push --force-with-lease origin main
   ```

5. **Notify team** that all keys have been rotated

## Verification Checklist

Before every push to GitHub:

- [ ] Run `git status` - no .env visible
- [ ] Run `git ls-files .env` - returns empty
- [ ] Check `.gitignore` includes `.env`
- [ ] Review what will be pushed: `git diff origin/main`

## For Production Deployments

**DO NOT use .env files in production!**

Use proper secret management:
- Windows: Credential Manager / Azure Key Vault
- AWS: Secrets Manager / Parameter Store
- GCP: Secret Manager
- Self-hosted: HashiCorp Vault
